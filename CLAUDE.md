# libdivecomputer — project notes for Claude

## What this repo is

Fork of [libdivecomputer](https://github.com/libdivecomputer/libdivecomputer) — a C library for
communicating with dive computers from various manufacturers (Suunto, Shearwater, Oceanic, Mares,
etc.). The upstream uses autotools; this fork adds a CMake build for Windows x64 with MinGW.

## Repository layout

```
CMakeLists.txt                  CMake build (Windows x64 / MinGW)
configure.ac                    Upstream autotools build
src/                            Library source (all .c files)
include/libdivecomputer/        Public headers
  version.h.in                  Template — version.h is generated by CMake
src/libdivecomputer.symbols     Exported symbol list (used to generate .def)
src/revision.h                  Gitignored — generated by CMake from git HEAD
deps/libusb/                    Submodule: libusb-1.0
deps/hidapi/                    Submodule: hidapi
contrib/msvc/                   Upstream MSVC project (reference only)
build/mingw64/                  CMake build output (gitignored)
```

## Toolchain

- **Compiler**: WinLibs MinGW-W64 GCC 16.x (UCRT) at `D:\work\winlibs\mingw64`
- **Build system**: CMake + MinGW Makefiles
- **Key binaries**: `gcc.exe`, `cmake.exe`, `windres.exe`, `mingw32-make.exe`
  all under `D:\work\winlibs\mingw64\bin\`

## Building

From `build\mingw64\` (create it if it doesn't exist):

```bat
D:\work\winlibs\mingw64\bin\cmake.exe -G "MinGW Makefiles" ^
  -DCMAKE_C_COMPILER=D:\work\winlibs\mingw64\bin\gcc.exe ^
  -DCMAKE_RC_COMPILER=D:\work\winlibs\mingw64\bin\windres.exe ^
  -DCMAKE_MAKE_PROGRAM=D:\work\winlibs\mingw64\bin\mingw32-make.exe ^
  ..\..

D:\work\winlibs\mingw64\bin\mingw32-make.exe -j4
```

CMake generates `version.h` (from `include/libdivecomputer/version.h.in`) and `revision.h`
(from `git rev-parse HEAD`) into the build directory at configure time.

## CMakeLists.txt design

- **libusb-1.0**: built as a static library directly from `deps/libusb/libusb/` sources.
  Requires a hand-written `config.h` (written to `build/libusb-config/config.h` at configure time)
  since libusb has no CMakeLists.txt.
- **hidapi**: added via `add_subdirectory(deps/hidapi)`. On Windows it builds `hidapi_winapi`
  (native Win32 HID API, no libusb dependency). Built as static.
- **libdivecomputer**: shared DLL. Both libusb and hidapi are statically linked in, so the DLL
  has no extra runtime dependencies beyond `ws2_32.dll` and standard Windows DLLs.
  Exports are controlled by `src/libdivecomputer.symbols` → `.def` file generated at configure time.

## Enabled transports

| Transport  | Active | Mechanism |
|------------|--------|-----------|
| Serial     | yes    | Win32 serial API |
| IrDA       | yes    | `af_irda.h` Windows IrDA sockets |
| Bluetooth  | yes    | `ws2bth.h` Windows RFCOMM sockets |
| USB        | yes    | libusb-1.0 (WinUSB/UsbDk, loaded dynamically) |
| USBHID     | yes    | hidapi Win32 backend (hid.dll, loaded dynamically) |
| BLE        | no     | No transport impl in this lib version; UUID helpers only |

## Key preprocessor defines

```
LIBDIVECOMPUTER_EXPORTS   DLL export marker
ENABLE_LOGGING            Enable internal logging
HAVE_VERSION_SUFFIX       Include git revision in version string
HAVE_AF_IRDA_H            IrDA support
HAVE_WS2BTH_H             Bluetooth support
HAVE_LIBUSB               libusb USB/USBHID support
HAVE_HIDAPI               hidapi USBHID support
```
